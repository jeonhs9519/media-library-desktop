## 릴리즈 개요

- 버전: `v0.0.7`
- 날짜: 2026-05-05
- 비교 기준: `v0.0.6`
- 성격: 문서 체계 정비, 검색/설정 UI 개선, 플레이리스트 구현, 릴리즈 자동화 보강

## 주요 기능

- 미사용 태그 정리와 태그 사용 건수 기반 필터링을 추가했습니다.
- 검색 조건 모달과 설정 모달 UI를 개선했습니다.
- 기본 플레이리스트 저장, 항목 추가, 제거, 재정렬, 뷰어 연동을 구현했습니다.
- PDF, CBZ, 비디오 뷰어에 플레이리스트 표시 토글, 이전/다음 항목 이동, 자동 다음 항목 실행을 연결했습니다.
- 과거 `media-library.db` 수동 가져오기와 HDT 가져오기 설정 흐름을 정리했습니다.

## UI/UX 개선

- 라이브러리 UI 구조와 접근성을 개선하고 roving focus, context menu, focus return 흐름을 정리했습니다.
- 페이지네이션을 하단 고정 영역으로 정리하고 한 페이지만 있어도 표시하도록 변경했습니다.
- 상세 삭제 확인을 `confirm()` 대신 공용 Modal로 변경했습니다.
- PDF, CBZ, 비디오 뷰어의 썸네일 업데이트 완료 알림을 `alert()` 대신 toast로 표시했습니다.
- 상세 정보의 파일 경로 표시와 파일 위치 열기 동작을 정리했습니다.
- 앱 startup 화면과 초기화 흐름을 개선해 창 표시와 라이브러리 목록 준비 시점을 앞당겼습니다.

## 구조 및 성능

- `docs/` 중심 문서 체계와 AI 작업 운영 문서를 정비했습니다.
- 라이브러리 화면의 컴포넌트, 모달, hook 분리를 마무리했습니다.
- `items` IPC를 core/relink/imports/metadata 세부 모듈로 분리했습니다.
- PDF, CBZ, 비디오 뷰어 route를 `React.lazy`로 분리하고 idle preload 흐름을 추가했습니다.
- `ffprobe-static` 패키징 바이너리 처리를 정리했습니다.

## 배포 및 CI

- Actions artifact 업로드 조건, 캐시, Node.js 버전, artifact quota 점검 워크플로를 정리했습니다.
- GitHub Actions 릴리즈 자동화에 테스트, 태그/버전 일치 검증, ZIP 검증, 릴리즈 노트 생성을 보강했습니다.
- 릴리즈 문서에 태그 기반 정식 릴리즈 경로와 로컬 검증 절차를 보강했습니다.
