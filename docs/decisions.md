# Decisions

프로젝트 운영과 문서화 방식에 영향을 주는 결정을 기록합니다.

## 2026-05-05

### Tauri 재구축은 기존 앱 완성 이후 진행

- 현재 Electron + React 버전을 먼저 기능적으로 완성합니다.
- 완성된 기존 앱을 Tauri 재구축의 기준 구현체로 사용합니다.
- Tauri 전환은 기존 프로젝트를 한 번에 변환하지 않고, 분석 문서와 재구축 계획서를 바탕으로 신규 프로젝트에서 단계적으로 진행합니다.
- 재구축 대상 기술 스택은 Tauri + Rust + Svelte를 우선 후보로 둡니다.

## 2026-05-02

### 플레이리스트 구현 완료 기준

- 플레이리스트 자체 기능의 잔여 구현 항목은 없는 상태로 봅니다.
- 플레이리스트 데이터는 현재 기본 목록으로 관리하고, 프로필별 분리 범위에 포함합니다.
- 라이브러리 화면에서는 플레이리스트 표시 위치가 좌측 또는 우측이어도 HTML 순서를 툴바, 플레이리스트, 라이브러리 본문, 페이지네이션 흐름으로 유지합니다.
- 라이브러리 화면의 플레이리스트는 라이브러리 본문 컬럼과 높이를 공유하고, 페이지네이션은 플레이리스트 접힘/펼침 여부와 관계없이 전체 폭으로 표시합니다.
- 플레이리스트 항목 재정렬과 라이브러리 항목 drop 추가는 삽입 위치 placeholder를 표시하며, 실제 순서가 바뀌지 않는 원래 위치 앞/뒤에는 placeholder를 표시하지 않습니다.
- 뷰어 플레이리스트는 현재 항목을 강조하고 현재 항목 제거를 비활성화합니다.
- 뷰어에서는 `L`로 플레이리스트 표시를 토글하고, `PgUp`/`PgDown`으로 이전/다음 플레이리스트 항목을 실행합니다.

## 2026-05-01

### 프로필 기능 도입과 완료 기준

- 프로필 기능은 도입합니다.
- 2026-05-06 기준 프로필 선택, 생성, 이름 변경, 전환, 삭제, 항목 이동/복사 기능은 현재 요구 범위에서 구현 완료로 봅니다.
- 개인이 개인 컴퓨터에서 사용하는 앱이라는 전제를 우선해 프로필 비밀번호는 제공하지 않습니다.
- `profiles` 테이블은 `id`, `name`, `createdAt`, `updatedAt`만 둡니다.
- `SYSTEM` 프로필의 `profiles.id` 값은 `1`로 고정하고, UI에는 표시하지 않습니다.
- `UNASSIGNED` 프로필의 `profiles.id` 값은 `2`로 고정하고, UI에는 표시하지 않습니다.
- `GUEST` 프로필의 `profiles.id` 값은 `3`으로 고정합니다.
- 사용자가 직접 생성한 프로필은 `profiles.id` 값 `4`부터 사용합니다.
- 프로필이 지정되지 않은 기존 데이터의 `profileId` 값은 `UNASSIGNED` 프로필 id인 `2`로 둡니다.
- 직접 생성 프로필 이름으로 `UNASSIGNED`, `SYSTEM`, `GUEST`는 사용할 수 없게 합니다.
- `settings`, `items`, `tags`, `playlists`는 `profileId`를 직접 가집니다.
- `settings` 중 `ui.language`, `video.volume`, `fileModifiedAt.updatePolicy`, `profile.lastActiveId`, `profile.lastActiveIds`, `profile.useLastOnStartup`은 `SYSTEM` 프로필에 귀속합니다.
- `settings` 중 `playlist.position`, `library.playlist.collapsed`, `viewer.playlist.visible`, `pdf.viewMode`, `cbz.viewMode`는 사용자 프로필에 귀속합니다.
- `UNASSIGNED`에 남아 있는 설정은 프로필 선택 시 `SYSTEM` 설정 키와 사용자 설정 키로 나눠 각각 `SYSTEM` 또는 선택된 프로필에 귀속하고, `UNASSIGNED` 설정 행은 남기지 않습니다.
- `itemTags`는 `items`와 `tags`의 연결 데이터로 보고 별도 `profileId`를 두지 않습니다.
- `playlistItems`는 `playlists`와 `items`의 연결 데이터로 보고 별도 `profileId`를 두지 않습니다.
- `reviews`는 `items`에 귀속되는 데이터로 보고 별도 `profileId`를 두지 않습니다.
- `items`는 파일 경로가 같더라도 프로필에 따라 별개 항목으로 취급합니다.
- `items`, `tags`, `playlists`, `settings`의 unique 제약은 전역 기준이 아니라 `profileId`를 포함한 프로필 범위 기준으로 조정합니다.
- 리뷰, 진행률, 플레이리스트, 설정을 포함한 모든 내용을 프로필별로 분리해 관리합니다.
- 최초 실행 시 프로필이 없으면 `GUEST`로 시작할지 신규 프로필을 생성할지 선택합니다.
- 기존 데이터가 있으면 최초 실행에서 선택한 프로필로 이관합니다.
- 최초 프로필 선택/생성 시 `profileId = 2`인 데이터가 있으면, 선택한 프로필로 해당 데이터가 귀속된다는 안내를 표시합니다.
- 최초 실행과 프로필 전환 시에는 앱 본문 진입 전 프로필 선택 화면을 표시합니다.
- 프로필이 지정되지 않은 데이터가 있을 때는 `※ 현재 프로필이 지정되지 않은 데이터가 존재합니다. GUEST 프로필로 시작할 경우 해당 데이터들은 GUEST 프로필에 귀속되며, 신규 프로필을 생성할 경우 생성된 프로필로 귀속됩니다.` 안내를 함께 표시합니다.
- 프로필 선택 화면에는 `GUEST`와 등록된 프로필 목록, 신규 프로필명 입력, 기존 데이터 이관 안내, `선택한 프로필로 시작` 버튼을 제공합니다.
- 기존 데이터 이관 안내는 `기존에 n개의 데이터가 존재하며, 선택된 프로필로 이관됩니다.` 형식으로 표시합니다.
- 프로필명 입력 placeholder는 `프로필명 입력`을 사용합니다.
- 프로필명은 최대 16자로 제한합니다.
- 프로필 선택 화면의 `다음 실행 시 이 프로필 사용` 체크 여부는 `SYSTEM` 설정인 `profile.useLastOnStartup`으로 저장합니다.
- `다음 실행 시 이 프로필 사용`이 체크되어 있으면 다음 앱 실행 시 `profile.lastActiveIds`의 첫 번째 유효 프로필로 바로 시작합니다.
- 프로필 선택 화면에서는 마지막으로 사용한 프로필명 오른쪽에 `마지막으로 사용`을 표시합니다.
- `마지막으로 사용` 문구 유무와 관계없이 프로필 항목 높이는 일정하게 유지하고, 프로필명 영역은 세로 중앙에 위치시킵니다.
- 설정 팝업 본문 최상단에 `프로필 관리` 항목을 추가합니다.
- 프로필 관리 항목에는 현재 프로필 이름 입력과 `이름 변경` 버튼을 제공합니다.
- `GUEST` 프로필에서는 이름 변경을 비활성화하고, `GUEST 프로필은 이름을 변경할 수 없습니다.` 안내를 표시합니다.
- 프로필 전환은 설정 팝업 footer 좌측 버튼으로 제공하며, 클릭 시 현재 프로필 선택을 해제하고 최초 실행 상태와 유사한 프로필 선택 화면으로 돌아갑니다.
- 신규 프로필 생성은 설정 팝업이 아니라 프로필 선택 화면에서 처리합니다.
- 프로필 선택 화면에는 `GUEST`와 등록된 프로필 목록, 신규 프로필명 입력, `선택한 프로필로 시작` 버튼을 제공합니다.
- 등록된 프로필 목록에는 마지막으로 선택한 프로필을 구분해 표시합니다.
- 프로필 삭제는 설정 팝업이 아니라 프로필 선택 화면에서 처리합니다.
- 프로필 삭제 시 확인 Modal을 표시하고, 삭제 대상 프로필의 데이터를 어느 프로필로 이관할지 또는 데이터와 함께 삭제할지 선택하게 합니다.
- 마지막 사용 프로필은 `profile.lastActiveIds`에 최대 2개 저장합니다. 첫 번째 프로필이 삭제되면 두 번째 프로필을 사용하고, 둘 다 사용할 수 없으면 `GUEST`를 사용합니다.
- 프로필 삭제 시 해당 프로필의 `playlists`는 삭제하며, `reviews`, `itemTags`, `playlistItems`는 명시적으로 삭제합니다.
- 프로필 삭제에서 `데이터 이관`과 `대상 프로필의 데이터 사용`을 선택하면 중복 항목은 대상 프로필의 항목을 유지하고 삭제 대상 프로필의 중복 항목과 부속 데이터를 삭제합니다.
- 프로필 삭제에서 `데이터 이관`과 `본 프로필의 데이터로 덮어쓰기`를 선택하면 대상 프로필의 중복 항목과 부속 데이터를 명시적으로 삭제하고, 대상 프로필 플레이리스트에 있던 중복 항목 참조는 이관 항목 id로 대체합니다.
- 프로필 삭제 이관 중 태그는 태그명 기준으로 대상 프로필의 태그를 재사용하고, 없으면 새로 생성합니다.

### 프로필 이관 UX

- 프로필 이관은 지정한 프로필로 관련 정보 일체를 이관합니다.
- 이관 대상에는 파일 정보, 진행률, 리뷰를 포함합니다.
- 라이브러리 목록의 썸네일 컨텍스트 메뉴와 상세 정보 모달에서 항목 이관을 제공합니다.
- `파일 위치 열기` 아래 구분선 다음에 `다음으로 이동` submenu를 표시합니다.
- `다음으로 이동` 하위 메뉴에는 `GUEST`를 포함한 프로필 목록을 표시합니다.
- `다음으로 이동` 아래에는 `다음으로 복사` submenu를 표시하며, 하위 메뉴 구성과 비활성화 기준은 이동과 동일하게 적용합니다.
- 같은 파일 경로, 파일명, 파일 확장자의 항목이 이미 존재하는 대상 프로필은 비활성화합니다.
- 항목 이관 시 기존 플레이리스트에서는 해당 항목을 제거하고, 대상 프로필의 플레이리스트에는 추가하지 않습니다.
- 항목 태그는 태그명 기준으로 대상 프로필의 태그를 재사용하거나 새로 생성한 뒤 다시 연결합니다.
- 항목 이관 완료 후 현재 프로필의 미사용 태그 정리를 실행합니다.
- 리뷰는 항목 id에 귀속되므로 별도 이관 처리를 하지 않습니다.
- 항목 복사 시 대상 프로필에 새 항목을 생성하고, 태그는 이름 기준으로 재사용 또는 생성해 연결하며, 리뷰도 새 항목에 복제합니다.
- 상세 정보 모달의 프로필 이동 항목은 리뷰 항목과 파일 항목 사이에 배치하며, 대상 프로필 select와 `이동`, `복사` 버튼으로 구성합니다.

### 과거 DB 가져오기와 프로필 귀속

- 과거 DB 가져오기는 설정 팝업에서 실행하므로 이미 active profile이 선택된 상태로 봅니다.
- 과거 DB에서 가져온 `items`, `tags`, `playlists`와 사용자 범위 `settings`는 `UNASSIGNED`가 아니라 현재 프로필에 귀속합니다.
- 과거 DB에서 가져온 설정 중 `SYSTEM` 설정 키는 `SYSTEM` 프로필에 귀속합니다.
- 과거 DB 가져오기 preview의 중복 항목, 태그, 플레이리스트, 설정 판정은 현재 프로필 범위를 기준으로 처리합니다.

### 플레이리스트 표시와 추가 UX

- 플레이리스트는 라이브러리 목록의 좌측 또는 우측에 표시할 수 있게 합니다.
- 기본 위치는 우측입니다.
- 위치 선택은 설정 팝업의 `표시 설정`에서 제공합니다.
- 최초 실행 시 플레이리스트는 라이브러리 목록 기준 `width: 32px; height: 100%;` 형태의 접힌 상태로 표시합니다.
- 접힘/펼침은 caret 버튼으로 토글합니다.
- 접힌 상태에서는 `<` 모양으로 펼치기를 유도하고, 펼쳐진 상태에서는 `>` 모양으로 접기를 유도합니다.
- 접힌 상태에서 리스트에 항목이 추가되면 자동으로 펼칩니다.
- 창 크기 변경뿐 아니라 접힘/펼침 상태 전환 시에도 라이브러리 목록 방향키 이동 계산을 갱신합니다.
- 항목 추가는 라이브러리 목록 컨텍스트 메뉴, 라이브러리 목록에서의 drag and drop, 상세보기 팝업 버튼으로 지원합니다.
- 상세보기 팝업의 `리스트에 추가` 버튼은 `뷰어 열기`와 `외부에서 열기` 사이에 배치합니다.
- 플레이리스트에는 `book`, `comic`, `video`만 추가할 수 있습니다.
- `other` 타입은 목록 컨텍스트 메뉴와 상세보기 버튼을 비활성화합니다.
- `other` 타입을 drag and drop할 때는 `cursor: disabled`를 적용합니다.

### 플레이리스트 자동 다음 항목 실행 조건

- `book`/`comic`은 마지막 페이지를 열람한 뒤 다음 페이지로 이동하려 할 때 다음 항목을 실행합니다.
- `video`는 100% 재생 완료 후 영상이 정지 상태가 되었을 때 다음 항목을 실행합니다.
- 다음 항목 실행은 상세보기 팝업에서 `뷰어 열기`를 선택할 때와 같은 방식으로 처리합니다.

### 상세 편집과 삭제 UX

- 편집과 삭제는 상세보기 팝업에서만 수행합니다.
- 삭제 시 복구할 수 없으며 새로 등록해야 함을 명시합니다.
- 삭제 확인은 `confirm()` 대신 별도 Modal을 사용합니다.

### 기본 대화상자 대체

- renderer에서는 `alert()`, `confirm()`, `prompt()`를 사용하지 않습니다.
- 확인이 필요한 작업은 공용 Modal을 사용합니다.
- 단순 완료 알림은 작업 흐름을 막지 않도록 toast를 사용합니다.
- 뷰어 썸네일 업데이트 완료 toast는 상단 툴바 바로 아래 우측에 표시합니다.
- toast는 사라질 때 나타날 때의 반대 방향 애니메이션을 사용합니다.

### UX 확인 완료 항목

- 뷰어 화면에서 `ESC`, `Backspace` 정상 동작을 확인했습니다.
- 라이브러리 목록과 상세 팝업의 포커스 이동 정상 동작을 확인했습니다.
- 검색 툴바, 페이지네이션, 설정 팝업의 `title` 정상 동작을 확인했습니다.

### Startup 준비 완료 기준

- 앱 시작 UX는 빈 창 또는 가벼운 로딩 화면을 먼저 표시하는 방향을 우선합니다.
- DB와 설정 조회 후 최초 라이브러리 목록이 화면에 표시되는 시점을 준비 완료 상태로 봅니다.
- 썸네일 일괄 로드는 준비 완료 이후 비동기로 진행합니다.
- PDF, CBZ, 비디오 뷰어 페이지는 초기 renderer 로드에서 분리하고, 목록 표시 이후 idle 시점에 미리 import하는 방향을 사용합니다.
- 이 결정은 startup 체감 개선과 첫 목록 표시 시간 단축을 우선하기 위한 기준입니다.
