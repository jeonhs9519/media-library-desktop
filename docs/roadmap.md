# Roadmap

Last reviewed: 2026-05-05

## 단기

- 프로필 기능 구현

## 중기

- 핵심 사용자 흐름 테스트 보강
- 상태 관리 단순화 또는 컴포넌트 분리 구조 안정화
- i18n 문구, toast, Modal 사용 패턴 정리

## 장기

- 앱 설정과 데이터 구조가 커질 경우 모듈 경계 재설계
- 라이브러리 규모가 커졌을 때 성능 병목 점검
- 문서와 코드 변경을 함께 유지하는 작업 습관 정착
- Electron 버전 안정화 이후 Tauri + Rust + Svelte 기반 재구축 검토
- 재구축 전 기존 앱의 기능 목록, 화면 목록, DB 스키마, IPC/API, 파일 처리 흐름을 문서화
- 신규 Tauri 프로젝트는 기존 코드 직접 변환이 아니라 명세 기반 단계적 재구축으로 진행

## 진행 원칙

- 구현 예정 아이템 전체를 한 번에 옮기지 말고, 실제 착수 가능한 순서대로 좁혀서 진행합니다.
- 구조 개선은 기능 추가 전에 필요한 최소 범위만 먼저 합니다.
- 완료된 항목은 `backlog.md`에서 제거하거나 `changelog.md`에 반영합니다.
