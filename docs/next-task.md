## 바로 시작하기 좋은 작업

### 1. Startup 확인

- PC 재부팅 직후 개발 앱 또는 패키징 앱을 실행합니다.
- startup 화면이 창 표시 직후 보이는지 확인합니다.
- main process console의 `[startup] ... done ...ms` 로그를 확인해 느린 단계를 기록합니다.
- 캐시가 따뜻한 상태에서는 startup 화면이 매우 짧게 지나갈 수 있습니다.
- 현재 확인 결과: 개발 앱은 정상 실행되었으나 이미 로딩이 빨라진 상태라 startup 화면을 눈으로 확인하기는 어려웠습니다.
- 다음 PC 재부팅 이후 가장 먼저 확인할 수 있도록 1순위를 유지합니다.

### 2. 플레이리스트 범위 정리

- 태그 필터링과 라이브러리 화면 분리 후속 작업은 완료되었으므로, 다음 기능 작업으로 진행합니다.
- 지원 대상: `book`, `comic`, `video`
- 비지원 대상(`other`) 차단 UX 정의
- 자동 다음 항목 재생/열람 트리거 조건 정리

### 3. 접근성 및 UX 잔여 항목

- 기능을 먼저 대강 마무리한 뒤 UX 마무리 단계에서 정리합니다.
- 완료:
  - 라이브러리 카드를 키보드 포커스 가능한 버튼으로 변경
  - `Enter`/`Space`로 상세 팝업 열기
  - 상세 팝업을 포함한 공용 `Modal` focus trap 추가
  - 라이브러리 카드 우클릭 컨텍스트 메뉴 추가
  - 컨텍스트 메뉴에 상세 정보, 뷰어 열기, 외부에서 열기, 출처 URL로 이동, 파일 위치 열기 추가
  - 출처 URL이 있는 항목은 목록 컨텍스트 메뉴에서도 출처 URL 열기가 활성화되도록 수정
  - 페이지네이션을 목록 하단 고정 영역으로 이동하고 페이지 번호 버튼을 제공
  - 첫 페이지/마지막 페이지의 이전/다음 버튼을 시각적으로 비활성화
  - 목록 포커스 진입 후 `Tab`/`Shift+Tab`으로 정상 탈출 가능하도록 수정
  - 상세 팝업을 닫으면 포커스가 썸네일 목록으로 복귀
- 다음 후보:
  - 카드 컨텍스트 메뉴에 편집/삭제 같은 관리 액션을 넣을지 결정
  - 뷰어 공통 단축키 `ESC`, `Backspace` 동작 재점검

### 4. 테스트 보강

- 기능을 먼저 마무리한 뒤 가장 후순위로 보강합니다.
- `items` 흐름에 대한 단위 테스트 또는 통합 테스트를 추가합니다.
- 최소한 다음 흐름은 자동 검증 대상으로 잡는 것이 좋습니다.
- 파일 추가
- 중복 파일 차단
- bulk relink 충돌 처리
- `.hdt` preview/apply
- 라이브러리 목록 roving focus와 상세 팝업 focus return
- 컨텍스트 메뉴의 출처 URL 열기 활성/비활성 조건
